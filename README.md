# GradCor: Gradient-Corridor Anycast Routing, Evaluated on Real Meshtastic Networks

Reproduction package for the paper **"An Empirically Calibrated Evaluation of Flood,
Source-Path, Next-Hop and Gradient-Corridor Forwarding on Real Meshtastic Networks"**.

GradCor is a proposed LoRa mesh routing method: one byte of state per active destination,
receiver-side anycast forwarding, no hop limit. The paper evaluates it — in both its original
age-widened form (v1) and its final clock-free form (GradCor-R) — against managed flooding,
stylized source routing and Meshtastic 2.6-style next-hop caching, by trace-driven replay on
production Meshtastic networks: a 7-day window of the Norwegian mesh plus five further
networks, each with its own empirically calibrated link model.

**All results, figures and caveats live in the paper**: read it at
[relaymonkey.github.io/mesh-routing-paper](https://relaymonkey.github.io/mesh-routing-paper/),
or locally as [`paper.html`](paper.html) / [`paper.pdf`](paper.pdf). This README describes
only the package; the result tables ship as CSVs and are regenerated by the scripts below.

## Contents

| Path | What it is |
|---|---|
| `paper.pdf`, `paper.html` | The paper (HTML is the source; PDF is rendered from it) |
| `sim_norway.py` | Trace-driven replay: link extraction, calibration, all five protocols |
| `experiment_widening.py` | Clock ablation (hazard curve, threshold sweeps, reactive/adaptive variants) |
| `experiment_asymmetry.py` | Resilience to gradients that underestimate true forward distance |
| `experiment_multinet.py` | Cross-network validation on six networks, per-network calibration |
| `experiment_plant.py` | Cold-start realism: broadcast-seeded / fallback planting vs the on-demand shortcut |
| `experiment_cancellation.py` | Managed-flood overhear-cancellation bias bound |
| `experiment_target99.py`, `experiment_custody.py` | Delivery ceiling + reliability ladder (retries, rescue flood, custody) |
| `experiment_hoplimit_multinet.py` | Broadcast hop-limit cost/benefit across all six networks |
| `experiment_broadcast.py` | Broadcast redundancy ladder (source/backbone/blanket/echo-triggered retransmission) |
| `experiment_alltoall.py`, `alltoall_results.csv` | All-to-all strategy-space sweep (who relays × persistence × source repeats) |
| `experiment_alltoall_multinet.py`, `alltoall_multinet.csv` | Same sweep on all six networks (curve slope vs evidence density) |
| `fig_*.py`, `plot_results.py` | Figure generators (all figures in the paper) |
| `data/` | Norway exports: traceroutes, hourly node activity, node roles |
| `data_networks/` | Same exports for Bay Area, Florida, SoCal, Portugal, Italia |
| `results_summary.csv`, `results_hourly.csv`, `multinet_results.csv`, `multinet_meta.csv` | Result tables |

## Reproduce

Python 3.10+; figures need `matplotlib`.

```bash
python3 sim_norway.py             # headline five-protocol replay (~20 s)
python3 experiment_widening.py    # clock ablation
python3 experiment_asymmetry.py   # asymmetry resilience
python3 experiment_multinet.py    # six-network validation (~2 min)
python3 experiment_plant.py       # cold-start realism (planting without the oracle)
python3 experiment_cancellation.py # flood overhear-cancellation bias bound
python3 experiment_target99.py    # delivery ceiling + reliability ladder
python3 experiment_custody.py     # temporal reachability + store-and-forward custody
python3 experiment_hoplimit_multinet.py # broadcast hop-limit sweep, six networks
python3 experiment_broadcast.py   # broadcast redundancy ladder incl. echo-triggered retransmit
python3 experiment_alltoall.py    # all-to-all strategy sweep (writes alltoall_results.csv)
python3 experiment_alltoall_multinet.py # same sweep, six networks (alltoall_multinet.csv)
python3 fig_paper.py && python3 fig_method.py && python3 fig_summary.py \
  && python3 fig_plant.py && python3 fig_ablation.py && python3 fig_multinet.py \
  && python3 fig_target99.py && python3 fig_alltoall.py
```

Render the PDF (any Chromium browser):

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless \
  --no-pdf-header-footer --print-to-pdf=paper.pdf paper.html
```

Everything is deterministic (fixed seeds); the scripts print the tables that appear in the
paper.

## Data notes

- Exports come from [RelayMesh](https://www.relaymonkey.com/relaymesh/)'s production ingestion of public Meshtastic MQTT traffic,
  window 2026-06-27 → 2026-07-04 (UTC).
- Ground truth is what MQTT gateways heard: absolute delivery rates are model outputs, not
  field measurements — rankings and ratios are the robust findings (see the paper's Threats to
  validity section).
- Node identifiers and names appear as broadcast publicly by the nodes themselves.
